# Repository Instructions

- For any design-related UI task, read `DESIGN.md` before making changes.
- Keep `DESIGN.md` and `design-tokens.json` synchronized one-to-one when editing design tokens.
- Do not introduce new colors, type sizes, spacing, radii, or shadows without updating both files and documenting the reason.
- For gameplay/UI design tasks, treat this product as a tell-driven multiplayer "not me" poker game.
- Preserve the read path: hero hand + community cards → opponent signals/tells → action.
- On mobile, do not hide opponent read information behind horizontal scrolling; all opponent read panels must stay visible.
- Use the primitive → semantic → component token architecture for all design-token edits.
