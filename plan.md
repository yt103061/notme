# カードインタラクションUI 設計プラン(Web版)

「ポケポケ(Pokémon TCG Pocket)のように、実際にカードを触っている感覚」を**Webベース**で実現するためのUI設計書。
あわせて、**ゲームの邪魔にならない範囲で「何ができるのか」「何が起こっているのか」が常に分かり、もっとプレイしたくなる**フィードバック体験を設計する。

本設計は2本柱で構成する:

- **柱A: 触感** — カードを物体として触っている感覚(1〜5章)
- **柱B: 伝達** — ゲームの変化が邪魔にならずに伝わり、次の一手を誘う演出(6〜8章)

---

## 1. 設計思想:カードは「画面上の絵」ではなく「物体」である

ポケポケが気持ちいい理由は、カードが**UI部品ではなく物理オブジェクトとして扱われている**ことに尽きる。分解すると次の5要素になる。

| 要素 | 現実のカード | Web UIでの再現手段 |
|------|-------------|-------------------|
| 質量・慣性 | 動かすと遅れてついてくる、離すと滑る | スプリング物理・速度の引き継ぎ |
| 厚み・剛性 | 傾けると面が回る、端を押すと沈む | CSS 3D(perspective)・押下位置に応じたチルト |
| 表面の光沢 | 傾けると光が走る、ホロが揺れる | 角度連動のグラデーション/虹色レイヤー |
| 触覚 | 持った・置いた・弾いた感触 | **音で代替**(Web Audio、後述)+ vibration API(対応環境のみ) |
| 空間の中の存在 | 影が落ちる、浮くと影が離れる | 高度(elevation)連動のドロップシャドウ |

### Webという土俵での勝ち筋

Webはネイティブに比べてハプティクスがほぼ使えない(iOS Safariは振動不可)。そのぶん**「音」と「視覚の物理整合性」に触覚の役割を背負わせる**。

- 音を「効果音」ではなく**触覚の代替チャネル**として設計する(5章)。紙の擦れ・着地・めくりを速度連動で鳴らす
- 遅延との戦い: Pointer Events + `touch-action: none` で既定のスクロール/ズームを奪い、タッチダウン反応を1フレーム以内に
- 120Hzディスプレイでは自動的に120fpsの追従になるよう、rAFループ+transform直書きで実装する

### 絶対原則(Non-negotiables)

1. **指(ポインタ)に吸い付く** — カードは常にポインタの真下にある。アニメーション再生中でも触れたら即座に指へ制御が移る(interruptible)。
2. **すべての動きはスプリング** — durationベースのイージングは使わない。速度を持った状態から自然に減衰する物理モデルのみ。
3. **離した瞬間が本番** — ドラッグ終了時の速度(velocity)を必ずスプリングに引き継ぐ。「ピタッと止まる」は禁止。
4. **触れた場所が意味を持つ** — カードの右端を押せば右が奥に倒れる。中央を押せば全体が浮く。ポインタ座標を使わない演出は作らない。
5. **60fps(可能なら120fps)を死守** — transform / opacity 以外をアニメーションさせない。ジェスチャー中にレイアウト・Reactの再レンダーを発生させない。
6. **マウスとタッチの両対応** — ホバーが存在するPCでは「触れる前の予感」(hover時の微チルト+光)を追加し、タッチでは省略する。同じ物理モデルで入力だけ差し替える。

## 2. カードの物理モデル

カード1枚を、以下の状態を持つ剛体としてモデル化する。すべてのインタラクションはこの状態を書き換えるだけで表現する。

```
CardPhysicsState {
  position:   { x, y }        // 画面上の位置
  elevation:  z               // 0=置かれている 〜 1=完全に持ち上げた
  rotation:   { rx, ry, rz }  // 3D回転(rx/ry=チルト, rz=平面回転)
  scale:      s               // 1.0 基準
  flip:       0 | 180         // 表裏
  velocity:   { vx, vy }      // 直近のポインタ速度
}
```

### 派生値(状態から自動計算されるもの)

- **影**: `elevation` から算出。
  - 置かれている時: `y-offset 2px, blur 6px, opacity 0.25`
  - 持ち上げた時: `y-offset 24px, blur 40px, opacity 0.35`(影がカードから「離れる」)
- **光沢(グレア)**: `rx, ry` から光源反射位置を算出。カード表面の放射グラデーションレイヤーが、チルトと逆方向に移動する。
- **ホロ効果(レア演出)**: 角度に応じて色相が回転する虹色レイヤー(`hue-rotate` + マスク + `mix-blend-mode: color-dodge`)。傾けたときだけ現れる。

### スプリング定数(基準値)

| 用途 | stiffness | damping | mass | 感触 |
|------|-----------|---------|------|------|
| ポインタへの追従 | 400 | 40 | 1 | 機敏、ほぼ遅延なし |
| 手を離した後の復帰 | 180 | 22 | 1 | ぷるんと1回オーバーシュート |
| チルトの戻り | 250 | 18 | 1 | 揺れながら水平に戻る |
| フリップ | 200 | 26 | 1 | 重みのある半回転 |
| 配る/並ぶ | 150 | 20 | 1 | ゆったり滑り込む |

「追従は硬く・復帰は柔らかく」の関係を維持する。

## 3. インタラクション・カタログ

### 3.1 タッチダウン(触れた瞬間)

最重要。「触れた瞬間に反応する」ことがすべての土台。

- `pointerdown` から **1フレーム以内** に開始:
  - `scale: 1.0 → 1.04`、`elevation: 0 → 0.3`
  - タッチ座標に応じたチルト: カード中心からのオフセットに比例して最大 **±8°**(押した側が奥に倒れる=実カードの端を押した挙動)
  - 音: ごく小さな紙の接触音
- 動かさず離した場合 → タップ(詳細表示へ)。動かした場合 → ドラッグへ遷移。
- PCでは `pointerenter` 時点で微チルト(±3°)+グレアをうっすら出す(「触れそう」の予感)。

### 3.2 ホールド&チルト(押さえたまま指を滑らせる)

- チルト量: ポインタ位置のカード中心からの正規化オフセット × 最大 **±12°**
- グレアレイヤーが逆方向へ流れ、ホロカードなら虹が走る
- 詳細ビューでは慣性付き自由回転に発展(3.7)

### 3.3 ドラッグ(持ち上げて運ぶ)

- 移動開始の閾値: **8px**。超えた瞬間:
  - `elevation: 0.3 → 1.0`、`scale: 1.04 → 1.08`、影が大きく柔らかく
  - 音: 紙を持ち上げる擦過音
- 追従はスプリング経由で、**わずかに遅れてついてくる**ことで質量を感じさせる
- **移動方向へのチルト**: 水平速度に比例して `rz` を最大 ±6°(空気抵抗でしなる表現)
- ドロップ可能領域に入ったら領域側が「受け入れ姿勢」を見せる(6.3)

### 3.4 リリース(離す)

- ポインタ速度を引き継いでスプリング着地:
  - 有効なドロップ先 → 吸い込まれるようにスナップ。着地の瞬間 `scale 1.08 → 0.98 → 1.0` の沈み込み + 影が縮む + 「タッ」という着地音
  - 無効な場所 → 元の位置へ弧を描いて帰る(投げた方向に一度膨らんでから戻る)
- **フリック**: リリース速度が閾値(1200px/s)超なら「投げる」と解釈(場に出す、手札に戻す等)

### 3.5 フリップ(裏返す)

- `ry` を 0→180° スプリング回転。90°を跨ぐ瞬間に表裏テクスチャを差し替え
- 回転中 `scale 1.0→1.06→1.0`(裏返すとき手前へ浮く)。90°通過時に紙が空を切る音

### 3.6 スタック/ファン(束と手札)

- **山札**: 下のカードを1pxずつずらして厚みを表現。最上段を引くと下がわずかに揺れる(束の摩擦)
- **手札(ファン)**: 弧に沿って `rz` を割り付け。カード間にポインタが入ると近傍カードが左右に逃げて隙間を作る
- 1枚引き抜くと残りが**抜いた位置から波及して**再整列(stagger 20ms)

### 3.7 詳細ビュー(カードを手に取って眺める)

- タップ → **カードそのものが**画面中央へ拡大移動(View Transitions API または FLIP で shared element化。別画面に差し替えない)
- 背景は暗転+ブラー(`backdrop-filter`)。ドラッグで慣性付き自由回転、離すと揺れながら正面へ
- モバイルWebでは `DeviceOrientation`(要許可)でジャイロ連動の光をオプション提供
- 下フリックで元の位置へ吸い込まれて戻る

### 3.8 パック開封(該当する場合)

- 横スワイプの軌跡に沿って封が裂ける(進捗はジェスチャー連動、途中で止まる)
- カードは**下から指で押し出すように**1枚ずつ登場。レアは登場前に光漏れで「気配」を出す

## 4. 視覚レイヤー構造(カード1枚の内部構成)

```
<div class="card-slot">            … perspective: 1000px の3D空間
  <div class="card-shadow"/>       … elevation連動、本体と独立に変形
  <div class="card-body">          … rotation/scale/flipが掛かる剛体
    <div class="face-back"/>       … 裏面(backface-visibility: hidden)
    <div class="face-front">
      <img class="artwork"/>
      <div class="holo-layer"/>    … 虹色(レアのみ、角度連動、color-dodge)
      <div class="glare-layer"/>   … 白ハイライト(全カード、角度連動、overlay)
      <div class="frame"/>         … 枠・テキスト(光沢の影響を受けない)
      <div class="state-ring"/>    … アフォーダンス用の縁光(6.2、普段は透明)
    </div>
  </div>
</div>
```

- 影を本体と分離することで「浮くと影が遅れて小さくなる」表現が可能になる
- glare/holo はチルト角のみを入力とする純粋な写像にする(状態を持たない)
- すべて `will-change: transform` + GPU合成レイヤー。ジェスチャー中はこのDOMサブツリーを一切再構築しない

## 5. 音の設計(Webにおける触覚の代替)

Webではハプティクスに頼れないため、**音がカードの触覚を担う**。「効果音」ではなく「接触音」として設計する。

| 物理イベント | 音 | 設計指針 |
|----------|-----|----------|
| タッチダウン | ごく短い紙の接触音 | -30dB程度。意識に上らない音量 |
| 持ち上げ | 紙の擦過音 | ドラッグ速度にピッチ連動 |
| 着地・スナップ | 「タッ」という紙の着地音 | 落下速度で音量が変わる |
| フリップ90°通過 | 紙が空を切る音 | |
| 手札を撫でる | パラパラ音(1枚ごと) | 撫でる速度にピッチ・間隔連動 |
| 無効操作 | 低く鈍い音 | 罰ではなく「壁に当たった」感触 |
| レア演出・勝利 | 専用SE | ここだけ「効果音」でよい |

- Web Audio API を使用(`<audio>` はレイテンシ不可)。初回ユーザー操作で `AudioContext` を resume するゲートを設ける
- **速度→音量・ピッチのマッピング**が生っぽさの核。固定サンプルの単発再生にしない
- `navigator.vibrate` が使える環境(Android Chrome等)では着地・持ち上げに短い振動を重ねる(プログレッシブエンハンスメント)
- 音・振動は設定で個別にオフ可能。ミュート時でも視覚だけで全情報が伝わること(6章の要件)

## 6. 伝達デザイン:「何ができるのか」が邪魔にならずに分かる

ここからが柱B。原則は **「ゲームの主役は盤面とカード。UIは語りすぎない」**。

### 6.1 注意のヒエラルキー(すべての演出はこの3段階のどれかに属する)

| レベル | 目的 | 手段 | 例 |
|--------|------|------|-----|
| **L1: 環境光** | 意識に上らない常時情報 | 彩度・明度の差、微小な呼吸アニメ | 出せるカードがわずかに明るい |
| **L2: 所作** | 見れば分かる変化 | カード自身の動き・光、300ms以内 | ドロー、ダメージ、ターン交代 |
| **L3: 見得(みえ)** | プレイを止めて見せる瞬間 | 画面全体の演出、1〜2秒 | 必殺技、勝敗、レア開封 |

**運用ルール**:
- L3は1ゲームに数回まで。乱発すると「邪魔」になる(スキップ/短縮も必ず用意)
- L1は「気づかれないこと」が成功条件。プレイヤーが説明できなくても正しい選択をしていればよい
- 迷ったら1段下げる。演出の既定値はL2

### 6.2 アフォーダンス:「何ができるか」の表現

テキストやチュートリアルではなく、**カードの物理状態で選択肢を語る**。

- **出せるカード**: 手札の中でわずかに浮いている(`elevation: 0.05`、影が少し濃い)+ 縁がごく薄く呼吸するように光る(state-ring、周期3s)。出せないカードは沈んで彩度-15%
- **今引けるもの**(山札など): ホバー/近接で最上段カードがわずかに首をもたげる
- **ドラッグ開始時**: そのカードを置ける場所すべてのスロットが**同時にほんのり明るくなる**(L1)。置けない場所は変化なし。「探させない」
- **ドロップ先に接近**: スロットが広がってカードを迎えにくる(スロット側が `scale 1.0→1.06`)+ カードの影がスロットに吸い寄せられる(L2)
- **コスト不足など条件未達**: 掴もうとすると持ち上がりが浅い(`elevation` が0.15で頭打ち)+ 鈍い音。**掴めること自体は許す**(なぜ出せないかをツールチップで示す機会になる)
- **PCのホバー**: 微チルト+コスト/効果のハイライト。タッチでは長押し(300ms)が同じ役割

### 6.3 フィードバック:「何が起こっているか」の表現

ゲーム状態の変化は、**変化した物そのものを動かして**伝える。別の場所に数字やログを出すのは補助にとどめる。

| ゲームイベント | 演出(レベル) | 詳細 |
|----------------|--------------|------|
| ドロー | L2 | 山札から手札へカードが弧を描いて飛ぶ。1枚ずつ、既存の手札が波及して開く |
| 相手のプレイ | L2 | 相手側から盤面へカードが**着地音付きで**置かれる。自分の操作を止めない |
| ダメージ | L2 | 受けた側のカードが衝撃方向にノックバック+短い揺れ。数値は当たった位置からふわっと浮いて消える |
| 破壊・退場 | L2 | 崩れ落ちる/裏返って捨て札へ滑っていく。捨て札の山がわずかに跳ねて受け止める |
| バフ/デバフ | L1+L2 | 付与の瞬間だけ光が走り(L2)、以後は縁色と数値バッジで常時表示(L1) |
| ターン交代 | L2 | 盤面全体の照明がひと呼吸ぶん自分側/相手側に傾く+自分の出せるカードが浮き直す。バナーで遮らない |
| 必殺・レア級イベント | L3 | 短い暗転+カメラ寄り。**タップでスキップ可** |
| 勝敗 | L3 | 唯一、盤面を完全に止めてよい場面 |

**運用ルール**:
- 同時多発イベントは**因果の順に直列再生**し、1件あたりを短く(200ms前後)。全部同時に光らせない
- 相手ターン中の変化もすべて同じ文法で再生する(自分が操作した時と同じ動き)。「見ていなかった間に盤面が変わっていた」を絶対に作らない
- **履歴の担保**: 直近の変化を遡れる小さなログ(スワイプで出るドロワー)を用意。演出を見逃しても情報は失われない
- 演出中もゲームは基本ロックしない。入力は受け付けてキューイングする(L3のみ例外)

### 6.4 「邪魔にならない」の判定基準

- 演出を**全部オフにしても勝てる**(情報は色・配置・数値で冗長化されている)
- 演出を**全部オンにしても操作待ちが発生しない**(自分の入力がアニメーション完了待ちでブロックされる場面がL3以外に存在しない)
- 連続プレイ時に鬱陶しくならない(同一演出の3回目以降は自動的に20%短縮する、等の逓減を入れる)

## 7. 「もっとプレイしたくなる」演出の設計

継続欲求は派手さではなく、**操作の快感 × 変化の見通し × 次の一手の誘い**から生まれる。

### 7.1 操作そのものを報酬にする

- 柱Aの触感がここに直結する。**何も起きなくても触っていたいカード**が土台
- 「置く」快感を最重視: スナップ着地の沈み込み+音+スロットの受け止めは、全操作の中で最も磨く
- 良い手を打った時ほど気持ちよく: 大型カードほど着地が重い(沈み込み深く、音が低い)。コンボ成立時は着地音が和音に変化する等、**プレイの質が触感に反映される**

### 7.2 次の一手を誘う

- 自分のターン開始時、出せるカードが波及順に「ふわっ」と浮き直す(選択肢の提示、L1〜L2)
- 何もせず数秒経ったら、有効なカードの呼吸光をわずかに強める(急かさない範囲のナッジ。設定でオフ可)
- ドラッグを始めた瞬間に置ける場所が示される(6.2)ため、「試しに掴んでみる」こと自体が探索行為として成立する。**掴んで戻すのはノーコストで気持ちいい**ようにする(戻りの弧を丁寧に)

### 7.3 進歩と山場を感じさせる

- ゲーム内の緊張度(残りライフ、山札の残量など)を**環境光で**表現: 終盤ほど盤面の照明がわずかにドラマチックに(コントラスト+5%程度。気づかれなくてよい)
- リーサル(勝ち筋)級の状況では、キーカードの呼吸光を強める(L2)。「今、何かできそうだ」という予感を作る
- 勝利時は**自分が最後に置いたカード**を主役に見得を切る(自分の意思決定が勝因だった、という物語を毎回作る)

### 7.4 セッションの締めと再訪の誘い

- 敗北時は短く、素早くリトライ導線へ(敗北演出を長くしない)
- 勝敗後のリザルトで、そのゲームの「ハイライト1手」をカードの動きでリプレイ(3秒以内)
- 「もう1回」ボタンは、カードを山札に戻す所作(=次のゲームの準備動作)として演出する

## 8. 視覚トーンの方針

- 盤面は**暗め・低彩度**に沈め、カードだけが彩度を持つ(カードが主役、という照明設計)
- アフォーダンスの光は白〜カード属性色の低彩度。**警告色(赤)はダメージと危険にのみ**予約する
- テキストUI(ボタン・メニュー)は盤面の外周に置き、ゲーム中は半透明で沈ませる。触れると浮き上がる(カードと同じ物理文法)
- すべてのUI要素がカードと同じスプリング・同じ影のルールに従う(世界の物理法則を1つにする)

## 9. 技術アーキテクチャ(Web)

| 責務 | 採用技術 | 理由 |
|------|----------|------|
| 入力 | Pointer Events + `touch-action: none` | マウス/タッチ/ペン統一、既定ジェスチャーの排除 |
| 物理・アニメ | 手書きスプリング(rAFループ)+ transform直書き | ライブラリのReact再レンダー経由を避け、1フレームの遅延も入れない。`framer-motion`は非ゲージ部分(メニュー等)のみ可 |
| 3D/光沢 | CSS `perspective` + `rotateX/Y` + グラデーションレイヤー | GPU合成のみで完結 |
| ホロ等の高度な光学 | 必要になったら WebGL(three.js/regl)のカード面のみ置換 | まずCSSで作り、不足したら1枚単位で差し替え |
| shared element遷移 | View Transitions API(fallback: FLIP) | 一覧⇄詳細の連続性 |
| 音 | Web Audio API | 低レイテンシ、ピッチ/ゲインのリアルタイム制御 |
| 振動 | `navigator.vibrate`(対応環境のみ) | プログレッシブエンハンスメント |
| 状態管理 | ゲーム状態(権威)と表示物理状態(演出)を**完全分離** | ゲームロジックは即時確定、演出は後追い再生。ラグや演出スキップがゲーム状態を壊さない |

### 実装ルール

1. アニメーション対象は `transform` / `opacity` / `filter`(合成可能なもの)のみ
2. ジェスチャー→物理状態→style書き込みは**Reactの外**(rAFループ内)で完結。Reactはカードの生成・破棄とゲーム状態のみを扱う
3. カード画像は事前デコード(`img.decode()`)・2x解像度上限でキャッシュ。ドラッグ中のデコードジャンク禁止
4. `prefers-reduced-motion` 尊重: チルト・オーバーシュート・L3演出を無効化した簡易モードを用意(情報の等価性は6.4で担保済み)
5. イベント→演出は**キュー機構**を通す(6.3の直列再生・逓減・スキップを一元管理)

## 11. 実装フェーズ

### Phase 1: カードの「存在感」(最優先・感触の8割が決まる)
- 物理モデル+rAFスプリングループ、タッチダウン反応、ホールドチルト、グレア、接触音の基盤
- **完了条件**: カード1枚を押して撫でるだけで気持ちいい(触って15秒遊べるか)

### Phase 2: 持ち運びと「置く」快感
- ドラッグ・リリース・フリック、影の高度連動、スロットの受け入れ姿勢、スナップ着地
- **完了条件**: 掴んで置くだけの操作を繰り返したくなる

### Phase 3: 集合の振る舞い
- 手札ファン(割り込み・再整列・出せるカードの浮き)、山札・捨て札
- **完了条件**: 手札を指で割る・引き抜く動きが実カードの手癖と一致する

### Phase 4: 伝達レイヤー(柱B)
- fxQueue、6.3のイベント演出一式、アフォーダンス光、ターン交代、履歴ドロワー
- **完了条件**: 音を消し初見の人に相手ターンを見せて、何が起こったか説明できる

### Phase 5: 眺める体験と見得
- 詳細ビュー(View Transitions+自由回転)、フリップ、ホロ、L3演出(必殺・勝敗)、リザルト
- **完了条件**: レアカードを傾けて眺める時間が自然に発生する

### Phase 6: 仕上げ
- 演出の逓減・スキップ、reduced-motion、低性能端末フォールバック、実機での感触・テンポ総調整

## 12. 品質基準(受け入れテスト)

数値で守るもの:

- タッチダウン反応: **16ms(1フレーム)以内**
- ジェスチャー中のフレーム落ち: 0(ミドルレンジ実機のモバイルブラウザで計測)
- すべてのアニメーションが**途中で触って奪える**(interruptibility 100%)
- L2演出は1件300ms以内、同時多発時も入力ブロックなし

官能評価(リリース判定):

- [ ] カードを意味なく撫でたくなる(Phase 1 の15秒テスト)
- [ ] 「置く」操作だけを繰り返したくなる
- [ ] 音を消しても、相手ターンに何が起こったか初見で説明できる
- [ ] 演出をすべてオンにした連戦10ゲームで「待たされた」と感じる場面がない
- [ ] ゲーム終了時、「もう1回」を押す手が自然に動く
