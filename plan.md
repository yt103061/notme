# カードインタラクションUI 設計プラン

「ポケポケ(Pokémon TCG Pocket)のように、実際にカードを触っている感覚」をゼロベースで実現するためのUI設計書。
既存UIは前提にせず、「カードとは何か」から設計し直す。

---

## 1. 設計思想:カードは「画面上の絵」ではなく「物体」である

ポケポケが気持ちいい理由は、カードが**UI部品ではなく物理オブジェクトとして扱われている**ことに尽きる。分解すると次の5要素になる。

| 要素 | 現実のカード | UIでの再現手段 |
|------|-------------|----------------|
| 質量・慣性 | 動かすと遅れてついてくる、離すと滑る | スプリング物理・速度の引き継ぎ |
| 厚み・剛性 | 傾けると面が回る、端を押すと沈む | 3D回転(perspective)・押下位置に応じたチルト |
| 表面の光沢 | 傾けると光が走る、ホロが揺れる | 角度連動のグラデーション/シアン反射レイヤー |
| 触覚 | 持った・置いた・弾いた感触 | ハプティクス(振動)の打ち分け |
| 空間の中の存在 | 影が落ちる、浮くと影が離れる | 高度(elevation)連動のドロップシャドウ |

この5要素を**すべてのカード操作に例外なく適用する**ことが本設計の核。1つでも欠けると「絵が動いているだけ」に戻る。

### 絶対原則(Non-negotiables)

1. **指に吸い付く(Direct Manipulation)** — カードは常に指の真下にある。アニメーション再生中でも指が触れたら即座に指へ制御が移る(interruptible)。
2. **すべての動きはスプリング** — durationベースのイージングは使わない。速度を持った状態から自然に減衰する物理モデルのみ。
3. **離した瞬間が本番** — ドラッグ終了時の速度(velocity)を必ずスプリングに引き継ぐ。「ピタッと止まる」は禁止。
4. **触れた場所が意味を持つ** — カードの右端を押せば右が沈む。中央を押せば全体が浮く。タッチ座標を使わない演出は作らない。
5. **60fps(可能なら120fps)を死守** — transform / opacity 以外をアニメーションさせない。レイアウト・再レンダーをジェスチャー中に発生させない。

## 2. カードの物理モデル

カード1枚を、以下の状態を持つ剛体としてモデル化する。すべてのインタラクションはこの状態を書き換えるだけで表現する。

```
CardPhysicsState {
  position:   { x, y }        // 画面上の位置
  elevation:  z               // 0=置かれている 〜 1=完全に持ち上げた
  rotation:   { rx, ry, rz }  // 3D回転(rx/ry=チルト, rz=平面回転)
  scale:      s               // 1.0 基準
  flip:       0 | 180         // 表裏
  velocity:   { vx, vy }      // 直近の指の速度
}
```

### 派生値(状態から自動計算されるもの)

- **影**: `elevation` から算出。
  - 置かれている時: `y-offset 2px, blur 6px, opacity 0.25`
  - 持ち上げた時: `y-offset 24px, blur 40px, opacity 0.35`(影がカードから「離れる」)
- **光沢(グレア)**: `rx, ry` から光源反射位置を算出。カード表面に置いた放射グラデーションレイヤーが、チルトと逆方向に移動する。
- **ホロ効果(レア演出)**: 角度に応じて色相が回転する虹色レイヤー(`hue-rotate` + マスク)。傾けたときだけ現れる。

### スプリング定数(基準値)

| 用途 | stiffness | damping | mass | 感触 |
|------|-----------|---------|------|------|
| 指への追従 | 400 | 40 | 1 | 機敏、ほぼ遅延なし |
| 手を離した後の復帰 | 180 | 22 | 1 | ぷるんと1回オーバーシュート |
| チルトの戻り | 250 | 18 | 1 | 揺れながら水平に戻る |
| フリップ | 200 | 26 | 1 | 重みのある半回転 |
| 配る/並ぶ | 150 | 20 | 1 | ゆったり滑り込む |

※実装ライブラリに合わせて換算するが、「追従は硬く・復帰は柔らかく」の関係は維持する。

## 3. インタラクション・カタログ

### 3.1 タッチダウン(触れた瞬間)

最重要。「触れた瞬間に反応する」ことがすべての土台。

- 触れて **1フレーム以内** に開始(press-inのdelayなし):
  - `scale: 1.0 → 1.04`、`elevation: 0 → 0.3`
  - タッチ座標に応じたチルト: カード中心からのオフセットに比例して最大 **±8°**(端を押すとその側が沈むのではなく、**押した側が奥に倒れる**=実カードの端を押した挙動)
  - ハプティクス: 軽い1発(iOS: `UIImpactFeedbackGenerator(.light)` 相当)
- 指を動かさず離した場合 → タップ(詳細表示へ)。動かした場合 → ドラッグへ遷移。

### 3.2 ホールド&チルト(押さえたまま指を滑らせる)

カードを押さえたまま指をカード上で動かすと、指の位置に合わせてカードがシーソーのように傾く。

- チルト量: 指位置のカード中心からの正規化オフセット × 最大 **±12°**
- 同時にグレアレイヤーが逆方向へ流れ、ホロカードなら虹が走る
- これが「カードを撫でて眺める」体験の核。詳細画面では慣性付きで自由回転できる(後述 3.7)

### 3.3 ドラッグ(持ち上げて運ぶ)

- 移動開始の閾値: **8px**。超えた瞬間:
  - `elevation: 0.3 → 1.0`、`scale: 1.04 → 1.08`、影が大きく柔らかく
  - ハプティクス: 「持ち上げた」中強度1発
- 追従はスプリング(stiffness 400)経由。完全な等速追従ではなく、**わずかに遅れてついてくる**ことで質量を感じさせる
- **移動方向へのチルト**: 水平速度に比例して `rz` を最大 ±6° 傾ける(カードが空気抵抗でしなる表現)。垂直方向は `rx` に反映
- ドロップ可能領域に入ったら、領域側が「受け入れ姿勢」(スロットが広がる/光る)を見せ、軽いハプティクス

### 3.4 リリース(離す)

- 指の速度を引き継いでスプリングで着地:
  - 有効なドロップ先 → 吸い込まれるようにスナップ。着地の瞬間 `scale 1.08 → 0.98 → 1.0` の沈み込み + 影が縮む + 「置いた」ハプティクス(重め)
  - 無効な場所 → 元の位置へ弧を描いて帰る(速度を引き継ぐので、投げた方向に一度膨らんでから戻る)
- **フリック**: リリース速度が閾値(1200px/s)超なら「投げる」と解釈。用途例: カードを場に出す、手札に戻す、ゴミ箱へ

### 3.5 フリップ(裏返す)

- トリガー: 詳細表示中の左右スワイプ、またはダブルタップ
- `ry` を 0→180° スプリング回転。**90°を跨ぐ瞬間に表裏のテクスチャを差し替え**、裏面は鏡像にならないよう反転補正
- 回転中は `scale` を一時的に 1.0→1.06→1.0(裏返すときに手前へ浮く)
- 90° 通過時にハプティクス1発(カードが指を離れて回る感触)

### 3.6 スタック/ファン(束と手札)

- **山札(スタック)**: 数枚分の厚みを、下のカードを 1px ずつずらし重ねて表現。最上段を引くと下のカードがわずかに追従して揺れる(束の摩擦)
- **手札(ファン)**: 弧に沿って `rz` を割り付け(例: 5枚なら -12°〜+12°)。カード間に指が入ると近傍カードが左右に逃げて隙間を作る(実カードを指で割る動き)
- 手札から1枚引き抜くと、残りが**スプリングで再整列**(全カード同時ではなく、抜いた位置から波及するstagger 20ms)

### 3.7 詳細ビュー(カードを手に取って眺める)

- カード一覧からタップ → **カードそのものが**画面中央へ拡大移動(別画面に差し替えない。同一オブジェクトが連続的に動く = shared element)
- 背景は暗転+ブラー。カードだけが浮いている状態
- ドラッグで自由回転(`rx`, `ry` に慣性付き)。離すとゆらゆら揺れながら正面へ戻る
- ここでグレア/ホロが最大限に発揮される。**ジャイロ連動**(端末を傾けると光が動く)をオプションで追加
- 下方向へのフリックで手札/一覧へ戻る(吸い込まれるように元の位置へ帰る)

### 3.8 パック開封(該当する場合)

- パックを横スワイプ → スワイプ軌跡に沿って封が裂ける(進捗はジェスチャー連動、途中で止めれば裂け目も止まる)
- 裂き切るとカードが1枚ずつ**下から指で押し出すように**現れる。上スワイプで1枚ずつ送る
- レアカードは登場前に光漏れ+強ハプティクスで「気配」を出す

## 4. 視覚レイヤー構造(カード1枚の内部構成)

```
<Card>  ← perspective: 1000px を持つ3D空間
 ├─ shadow          … elevationに連動、カード本体とは独立に変形
 └─ card-body       … rotation/scale/flipが掛かる剛体
     ├─ face-back   … 裏面(backface-hidden)
     └─ face-front  … 表面(backface-hidden)
         ├─ artwork      … カード絵柄
         ├─ holo-layer   … 虹色レイヤー(レアのみ、角度連動、blend: color-dodge)
         ├─ glare-layer  … 白色ハイライト(全カード、角度連動、blend: overlay)
         └─ frame        … 枠・テキスト(光沢の影響を受けない)
```

- 影は本体と分離することで「浮くと影が遅れて小さくなる」表現が可能になる
- glare/holo はタッチ・チルト角のみを入力とする純粋関数にする(状態を持たない)

## 5. 触覚・音の設計

視覚だけでは「触っている感」の半分しか出ない。ハプティクスと音は**物理イベントに1:1対応**させる。

| イベント | ハプティクス | 音 |
|----------|--------------|-----|
| タッチダウン | light | なし(音は出しすぎない) |
| 持ち上げ(ドラッグ開始) | medium | 紙が擦れる短い音 |
| ドロップ可能領域に進入 | light | なし |
| 着地・スナップ | heavy(短) | 「タッ」という紙の着地音 |
| フリップ90°通過 | light | 紙が空を切る音 |
| 手札のカード間を撫でる | selection(カード1枚ごと) | パラパラ音(速度連動ピッチ) |
| レア演出 | heavy → light 連打 | 専用SE |

- 音量は控えめに。**ピッチと音量を操作速度に連動**させると生っぽさが跳ね上がる
- 設定でハプティクス/音を個別にオフにできること

## 6. 技術アーキテクチャ

### 前提

リポジトリは現状空のため、スタックは未確定。**モバイルネイティブ(React Native / Expo)を第一候補**、Webは第二候補として両対応の設計を示す。

### 第一候補: React Native (Expo)

| 責務 | 採用技術 | 理由 |
|------|----------|------|
| ジェスチャー | `react-native-gesture-handler` | UIスレッドでタッチ処理、閾値・速度取得が容易 |
| アニメーション | `react-native-reanimated` | worklet=UIスレッド駆動でJSブロックの影響を受けない。スプリング標準装備 |
| グレア/ホロ描画 | `@shopify/react-native-skia` | シェーダーで角度連動グラデーションを1パス描画 |
| ハプティクス | `expo-haptics` | 打ち分けAPIが揃っている |
| ジャイロ | `expo-sensors` | 詳細ビューの光連動用 |

**鉄則**: ジェスチャー→物理状態→スタイルの経路をすべてworklet内で完結させ、Reactの再レンダーを1フレームも挟まない。

### 第二候補: Web(PWA)

- Pointer Events + `framer-motion`(または手書きスプリング)
- 3Dは CSS `transform: perspective() rotateX() rotateY()`、グレアは `radial-gradient` の `background-position` を角度連動
- ハプティクスは `navigator.vibrate`(Android系のみ)。iOS Safariでは触覚が使えないため、**Webを主戦場にするなら視覚と音で補う**割り切りが必要
- この制約が「実際に触っている感覚」の目標に対して重いため、ネイティブを推奨する

### 共通の実装ルール

1. アニメーション対象は `transform` / `opacity` / (Skiaシェーダーuniform) のみ
2. カード画像は事前デコード・適切な解像度でキャッシュ(ドラッグ中のデコードジャンク禁止)
3. 物理状態 → スタイルの変換は派生値(第2章)として一元化。個別コンポーネントで勝手に影を計算しない
4. `prefers-reduced-motion` / OSの視差軽減設定を尊重し、チルト・オーバーシュートを無効化した簡易モードを用意(機能は等価に保つ)

## 7. コンポーネント設計

```
core/
  cardPhysics.ts      … CardPhysicsState、スプリング定数、派生値計算(影/グレア角)
  haptics.ts          … イベント名→触覚パターンのマッピング(打ち分けの一元管理)
  sound.ts            … イベント名→SE(速度連動ピッチ)

components/
  Card/
    Card.tsx          … レイヤー構造(4章)を組む純粋表示コンポーネント。物理状態を受け取るだけ
    GlareLayer.tsx    … 角度→ハイライト位置
    HoloLayer.tsx     … 角度→虹色シフト(レアのみ)
  gestures/
    usePressTilt.ts   … 3.1〜3.2(タッチダウン+チルト)
    useDragCard.ts    … 3.3〜3.4(ドラッグ+リリース+フリック)
    useFlip.ts        … 3.5
    useFreeRotate.ts  … 3.7(詳細ビューの自由回転+ジャイロ)
  layouts/
    CardFan.tsx       … 手札の弧配置+割り込み+再整列
    CardStack.tsx     … 山札の厚み+束の揺れ
    CardGrid.tsx      … 一覧(各セルに usePressTilt を適用)
  screens/
    CardDetail.tsx    … shared element遷移+自由回転+フリップ
    PackOpening.tsx   … 開封シーケンス(該当する場合)
```

設計上のポイント:

- **Card は物理状態を受け取るだけの純粋表示**。ジェスチャーはフックとして外付けし、同じCardが一覧・手札・詳細・開封で使い回せる
- ハプティクス・音を各所で直接呼ばず、`haptics.ts` / `sound.ts` の**イベント辞書経由**にする。感触の全体調整を1ファイルで行えるようにするため

## 8. 実装フェーズ

### Phase 1: カードの「存在感」(最優先・ここで感触の8割が決まる)
- CardPhysicsState と派生値計算
- タッチダウン反応(3.1)+ホールドチルト(3.2)+グレアレイヤー
- ハプティクス基盤
- **完了条件**: カード1枚を押して撫でるだけで気持ちいいこと。触って15秒遊べるか、を判定基準にする

### Phase 2: 持ち運び
- ドラッグ・リリース・フリック(3.3〜3.4)、移動チルト、影の高度連動
- ドロップ領域との連携
- **完了条件**: 投げて戻る動きが「紙が滑る」ように見えること

### Phase 3: 集合の振る舞い
- 手札ファン、スタック、抜き差し時の波及再整列(3.6)
- **完了条件**: 手札を指で割る・引き抜く動きが実カードの手癖と一致すること

### Phase 4: 眺める体験
- 詳細ビュー(shared element+自由回転+ジャイロ)、フリップ、ホロレイヤー(3.5, 3.7)
- **完了条件**: レアカードを傾けて眺めるだけの時間が発生すること

### Phase 5: 演出と仕上げ
- パック開封(3.8)、音、reduced-motion対応、低性能端末のフォールバック
- 実機での感触チューニング(スプリング定数・ハプティクス強度の総調整)

## 9. 品質基準(感触の受け入れテスト)

数値で守るもの:

- タッチダウン反応: **16ms(1フレーム)以内**
- ジェスチャー中のフレーム落ち: 0(実機ミドルレンジ端末で計測)
- すべてのアニメーションが**途中で触って奪える**こと(interruptibility 100%)

官能評価(リリース判定):

- [ ] 目を閉じて操作しても、振動だけで「持った/置いた/めくった」が区別できる
- [ ] カードを意味なく撫でたくなる(=Phase 1 の15秒テスト)
- [ ] 投げたカードの軌道に「重さ」を感じる
- [ ] 動画に撮って0.5倍速再生しても、姿勢・影・光の整合性が破綻していない
